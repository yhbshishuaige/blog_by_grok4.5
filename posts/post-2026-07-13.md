pi的文件结构
~~~shell
$tree ~/.pi
~/.pi
└── agent
    ├── auth.json
    ├── bin/
    ├── models.json
    ├── sessions/
    └── settings.json
~~~
相比于其他的AI coding agent,结构简单很多,这也是pi的核心思想之一,轻量,完全可定制

> 一个命令行 AI 编程工具(≈ codex cli),npm 装一下、配个 models.json,就能拿它当统一入口去调各种大模型;换模型一个 /model搞定,不用反复登录;而且因为内核很轻,所以启动快、省 token。

## auth.json

/login登录的信息, 不可删除! 里面的key需要有效, 不是单纯的记录文件或日志

~~~json
{
    "deepseek": {
        "type": "api_key",
        "key": "sk-**********************"
    }
}
~~~

/login会自动将登录信息保存到auth.json文件中, 这意味着一般不需要手动编辑auth.json文件

- type有两种

~~~text
 Select authentication method:

 → Use a subscription
   Use an API key
~~~

## models.json

pi支持自建provider, 自建provider的信息放在models.json中

- 也就是兼容大部分第三方api平台, 少部分限制了客户端, 无法使用pi
    - 但也可以通过添加"Header"来伪造, 比如可以伪造codex cli
~~~json
"headers": { "User-Agent": "codex_cli_rs/0.144.0" },
~~~

格式如下,举例: newapi+gpt

~~~json
{
    "providers":{
        "first_procider": {
            "baseUrl": "https://api.example1.com/v1",
            "api": "openai-completions",
            "apiKey": "sk-**********************",
            "reasoning": true,
            "models": [
                {
                    "id": "gpt-5.6-sol",
                    "name": "first_model",
                    "thinkingLevelMap": {
                        "minimal": null,
                        "low": "low",
                        "medium": "medium",
                        "high": "high",
                        "xhigh": "xhigh",
                        "max": "max"
                    },
                    "contextWindow": 128000,

                },
                { "id": "gpt-5.5" }
            ]
        },
        "second_provider": {
            "baseUrl": "https://api.example2.com/v1",
            "api": "openai-completions",
            "apiKey": "sk-**********************",
            "models": [
                { "id": "gpt-5.6-sol" } ,
                { "id": "got-5.5" }
            ]
        }
    }
}
~~~

添加好之后直接可以使用/model进行切换, 不需要/login, 即不需要向auth.json中添加

- 如果auth.json中有相关信息, 需要保证auth.json和models
.json中的信息保持一致,否则无法正常使用

**参数说明**

1. "api": 选择api协议, 官方文档中提供了四个协议
![a](img/pi_agent_apis.png)
    - openai-completions是兼容性最高的, 一般选择这个

    - openai-responses openai推出的,一般用于gpt系列模型

    - anthropic-messages a社专属,一般用于claude系列模型

    - google-generative-ai google推出,一般用于gemini系列模型

2. "reasoning": 思考深度开关,开启之后可以使用shift+tab进行思考深度切换

3. "models":
    1. "name": 使用/model显示的名称
    2. "thingLevelMap": 自定义各个模型的思考强度,如gpt系列有high,xhigh,max等等, 可以使用shift+tab进行切换
    3. "contextWindow": 上下文窗口


## settings.json

pi agent的设置文件, /settings可以快速编辑

~~~json
{
  "lastChangelogVersion": "0.78.0",
  "defaultProvider": "first_provider",
  "defaultModel": "gpt-5.6-sol",
  "defaultThinkingLevel": "high",
  "theme": "dark"
}
~~~
